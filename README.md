## ✨ Features

- **Real-time Face Recognition** — Auto-detect & match faces using DeepFace (ArcFace + RetinaFace)
- **Automated Attendance Logging** — Log entry/exit with confidence scores and camera source
- **Admin Dashboard** — Live stats, recent scans, camera feeds, one-click actions
- **Face Registration** — Webcam capture, submit for approval, admin approval workflow
- **Attendance Management** — Paginated logs, search, filter, date range, CSV export
- **Notification System** — Real-time Socket.IO notifications for admins and users
- **Role-based Access Control** — Admin and Employee roles with protected routes
- **JWT Authentication** — Access tokens (15 min) + refresh tokens (7 day HttpOnly cookie)
- **Password Recovery** — Email-based password reset flow (Nodemailer + Ethereal)
- **Responsive UI** — Dark/blue theme, mobile-friendly sidebar, tailwind animations

## 🏗️ Architecture

```
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│   Frontend    │────▶│   Backend     │────▶│  AI Service   │
│ React + Vite  │     │ Express + TS  │     │ FastAPI       │
│  Port 3000    │     │  Port 5000    │     │  Port 8000    │
└───────────────┘     └───────┬───────┘     └───────────────┘
                              │
                    ┌─────────┴─────────┐
                    │                   │
              ┌─────┴─────┐      ┌─────┴─────┐
              │ PostgreSQL │      │   Redis   │
              │ Port 5432  │      │ Port 6379 │
              └───────────┘      └───────────┘
```

- **Frontend** → Calls only the Backend API (`/api/*`)
- **Backend** → Acts as API gateway, calls AI service internally
- **AI Service** → Face embedding extraction + recognition via DeepFace
- **PostgreSQL** → Primary data store via Prisma ORM
- **Redis** → Session caching & real-time pub/sub

## 🚀 Getting Started

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/) & [Docker Compose](https://docs.docker.com/compose/install/)
- Ports `3000`, `5000`, `8000`, `5432`, `6379` available

### Quick Start

```bash
# 1. Clone & navigate
cd watchguard

# 2. Copy environment file (already provided, review & edit if needed)
cp .env.example .env

# 3. Start all services
make dev
# or
docker compose up --build

# 4. Wait for all services to be healthy, then run migrations & seed
make migrate
make seed

# 5. Open the app
# Frontend: http://localhost:3000
# Backend API: http://localhost:5000/api
# AI Health: http://localhost:8000/health
```

## 📡 API Reference

### Authentication

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| `POST` | `/api/auth/login` | Login with email & password | ❌ |
| `POST` | `/api/auth/refresh` | Refresh access token | Cookie |
| `POST` | `/api/auth/logout` | Logout & clear cookies | ✅ |
| `GET` | `/api/auth/me` | Get current user | ✅ |
| `POST` | `/api/auth/forgot-password` | Send password reset email | ❌ |
| `POST` | `/api/auth/reset-password` | Reset password with token | ❌ |

### Face Recognition

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| `POST` | `/api/face/recognize` | Recognize face (file or base64) | ✅ |
| `POST` | `/api/face/register` | Submit face registration request | ✅ |

### Users (Admin)

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| `GET` | `/api/users` | List all users (paginated) | Admin |
| `GET` | `/api/users/:id` | Get user by ID | ✅ |
| `PUT` | `/api/users/:id` | Update user | ✅ |
| `DELETE` | `/api/users/:id` | Soft-delete user | Admin |

### Attendance

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| `GET` | `/api/attendance` | List logs (paginated, filtered) | ✅ |
| `GET` | `/api/attendance/stats` | Dashboard statistics | Admin |
| `GET` | `/api/attendance/export` | Export logs as CSV | Admin |
| `GET` | `/api/attendance/:userId/history` | User attendance history | ✅ |

### Approvals (Admin)

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| `GET` | `/api/approvals` | List pending face requests | Admin |
| `POST` | `/api/approvals/:id/approve` | Approve request | Admin |
| `POST` | `/api/approvals/:id/reject` | Reject request | Admin |

### Cameras (Admin)

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| `GET` | `/api/cameras` | List all cameras | ✅ |
| `POST` | `/api/cameras` | Create camera | Admin |
| `PUT` | `/api/cameras/:id` | Update camera | Admin |

### Notifications

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| `GET` | `/api/notifications` | List user notifications | ✅ |
| `PUT` | `/api/notifications/:id/read` | Mark as read | ✅ |
| `PUT` | `/api/notifications/read-all` | Mark all as read | ✅ |

### Socket.IO Events

| Event | Direction | Description |
|-------|-----------|-------------|
| `face:detected` | Server → Client | New face recognized |
| `attendance:update` | Server → Admin | Periodic attendance refresh |
| `notification:new` | Server → Client | New notification |
| `approval:new` | Server → Admin | New face request submitted |

## 💻 Development

### Makefile Commands

```bash
make dev          # Start all services (attached)
make dev-detached # Start in background
make build        # Build all containers
make stop         # Stop all services
make clean        # Stop & remove volumes
make migrate      # Run Prisma migrations
make seed         # Seed database
make logs         # View logs
make studio       # Open Prisma Studio
```

### Running Individually

```bash
# Backend only
cd backend && npm install && npm run dev

# Frontend only  
cd frontend && npm install && npm run dev

# AI Service only
cd ai-service && pip install -r requirements.txt && uvicorn main:app --reload --port 8000
```

### Database Management

```bash
# Generate Prisma client
cd backend && npx prisma generate

# Create migration
cd backend && npx prisma migrate dev --name <name>

# Reset database
cd backend && npx prisma migrate reset

# Open Prisma Studio (GUI)
cd backend && npx prisma studio
```
